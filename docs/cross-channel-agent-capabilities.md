# 跨渠道助理能力

日期：2026-07-01

## 核心想法

产品不应该只成为一个新的聊天工具。它应该成为用户的跨渠道工作代理层：用户把邮件、钉钉、飞书、企业微信、Slack、Teams 等账号授权给个人助理后，助理可以统一接收、理解、整理、处理和回复这些渠道里的工作消息。

目标不是把所有外部平台立刻替换掉，而是先接管用户每天被打断的消息入口。

## 账号连接

用户可以连接：

- 邮件账号：Gmail、Outlook、企业邮箱、IMAP/SMTP。
- 企业 IM：钉钉、飞书、企业微信、Slack、Teams。
- 日历：Google Calendar、Outlook Calendar、企业日历。
- 任务工具：Linear、Jira、Asana、飞书任务、钉钉待办等。
- 文档和知识库：Notion、飞书文档、Confluence、Google Drive、企业网盘。

每个连接都需要明确：

- 助理能读什么。
- 助理能搜索什么。
- 助理能不能写入。
- 助理能不能发送消息。
- 哪些动作需要用户确认。
- 哪些敏感内容禁止跨渠道使用。

## 消息接管能力

助理接入外部渠道后，应能做这些事：

1. 统一收件：把不同渠道的消息聚合到一个工作入口。
2. 去重归并：同一件事在邮件、IM、会议里重复出现时，归并到同一话题。
3. 识别优先级：判断哪些消息需要立即处理，哪些可以稍后看。
4. 提取任务：从消息里提取请求、承诺、截止时间、待跟进、风险和决策。
5. 生成话题：把散落消息组织成话题，保留上下文和参与人。
6. 草拟回复：根据用户语气、历史上下文和权限规则生成回复草稿。
7. 自动回复：对低风险、明确规则的消息自动回复。
8. 升级给人：遇到高风险、不确定或敏感内容时，提醒用户确认。

## 代执行能力

用户可以对助理说：

- “帮我给张三发封邮件，说明明天下午没法参加会议。”
- “帮我回复这个飞书消息，说我今晚前给他方案。”
- “把这封邮件整理成 todo，明天上午提醒我。”
- “把客户邮件同步到项目话题里，并让销售 Agent 跟进。”
- “找一下上周王总在钉钉里提到的预算问题。”
- “给项目群发一条更新，但不要提客户报价。”

助理可以执行：

- 发邮件。
- 回复 IM。
- 转发或同步消息。
- 创建日历邀请。
- 创建/更新 todo。
- 创建/更新话题。
- 调用其他 Agent。
- 查询历史消息。
- 生成文档或纪要。
- 发起审批。

## 出站动作分级

助理能不能直接发出去，取决于风险等级。

### 低风险：可自动执行

- 确认收到。
- 发送常规模板回复。
- 预约已授权时间段。
- 创建 todo。
- 把消息归并到话题。
- 生成摘要。

### 中风险：默认草拟，用户确认后发送

- 对同事承诺交付时间。
- 修改项目任务状态。
- 发送会议调整。
- 给客户发送非敏感信息。
- 让其他 Agent 执行有副作用的动作。

### 高风险：必须确认

- 对外报价。
- 法务、合同、薪资、绩效、财务、客户机密。
- 对外承诺资源或时间。
- 代表公司立场发言。
- 跨渠道转发敏感内容。

### 禁止自动执行

- 转账、付款、签署合同。
- 删除重要数据。
- 授权新权限。
- 泄露他人隐私。
- 绕过原平台安全策略。

## 渠道差异

不同渠道的沟通语境不同，助理必须区别处理。

### 邮件

- 更正式。
- 可承载长文本和附件。
- 更常用于对外沟通。
- 发送前更应审慎。

### IM

- 更即时。
- 上下文更碎。
- 更适合快速确认、催办和同步状态。
- 需要避免在群里泄露私密上下文。

### 团队内部话题

- 更适合沉淀工作上下文。
- 可以承载任务、决策、文件和 Agent 执行记录。
- 助理应把外部消息归并到内部话题，而不是只转发。

## 核心产品价值

跨渠道接管不是“消息聚合器”，而是让助理拥有工作处理权：

- 用户不再需要轮流打开邮件、钉钉、飞书和其他 IM。
- 助理先判断消息是不是重要。
- 助理把消息转成话题、任务和待确认动作。
- 助理能处理低风险沟通。
- 用户只处理需要本人判断的部分。

## MVP 建议

第一版不应同时深接所有平台。建议按价值和可实现性分阶段：

1. 先接邮件：Gmail/Outlook/IMAP + SMTP。邮件标准化更好，适合验证助理代发、草拟、提任务和敏感内容控制。
2. 再接一个团队 IM：优先 Slack 或飞书，验证群聊、话题归并和 IM 回复。
3. 再接日历：让助理完成约时间和会议协调。
4. 最后接任务/文档工具：让任务执行结果能落到外部系统。

## 开放问题

- 第一优先接入哪个渠道：邮件、飞书、钉钉、企业微信、Slack，还是用户手动转发消息？
- 助理默认可以自动回复哪些类型的消息？
- 用户是否愿意把外部账号授权给这个产品？
- 企业管理员是否需要统一控制哪些渠道可以接入？
- 同一个人在不同渠道的身份如何合并？
- 外部消息归并到内部话题时，如何避免隐私泄露？
