# 任务工具栏控件设计

## 背景

任务工具栏中的搜索输入框和搜索按钮当前被做成紧贴的组合控件，并移除了相邻侧圆角。输入框使用默认 `h-9`，三个筛选按钮和搜索按钮使用 `size="sm"` 的 `h-8`，导致同一行控件高度不一致。

## 目标

- 状态、优先级、负责人、搜索输入框和搜索按钮位于同一个工具组。
- 工具组使用统一的 `gap-2` 间距。
- 搜索输入框和搜索按钮是两个独立控件，各自保留完整圆角。
- 所有控件使用默认 `h-9` 高度。

## 实现

修改 `client-web/src/components/projects/project-tasks-tab.tsx` 的 `TaskToolbar`：

- 移除包裹搜索输入框和按钮的无间距 flex 容器。
- 保留搜索图标所需的输入框定位容器，并让它直接成为外层 `gap-2` 工具组的子项。
- 从输入框移除 `rounded-r-none`。
- 从搜索按钮移除 `rounded-l-none` 和 `size="sm"`。
- 从三个 `FilterButton` 移除 `size="sm"`。

`Input` 和默认尺寸的 `Button` 均为 `h-9`，因此不增加应用侧高度覆盖，也不修改 `components/ui`。

## 非目标

- 不实现筛选或搜索功能。
- 不改变任务视图切换器。
- 不修改工具栏之外的控件尺寸。
- 不新增前端测试。

## 验证

- 五个控件位于同一 `gap-2` 工具组中。
- 搜索输入框和搜索按钮之间有 8px 间距，且各自有完整圆角。
- 输入框、筛选按钮和搜索按钮高度均为 36px。
- 运行前端现有测试、类型检查、Lint 和构建。
