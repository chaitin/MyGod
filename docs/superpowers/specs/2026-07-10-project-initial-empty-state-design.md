# 项目初始空状态设计

## 背景

项目页面首次进入时会自动选中第一个协作项目，与消息页面和联系人页面首次进入时不选择列表项的行为不一致。

## 目标

- 首次进入项目页面时不选中任何项目。
- 项目列表初始不显示选中态。
- 右侧显示居中的弱化文案“选择一个项目查看详情”。
- 用户点击项目后继续显示现有项目详情面板。

## 实现

将 `activeProjectId` 改为可空状态并以 `null` 作为初始值。当前项目根据该 ID 查找；没有 ID 或找不到对应项目时返回 `null`，不再回退到第一个协作项目。

右侧根据当前项目分支渲染：存在项目时渲染现有 `ProjectPanel`，否则渲染独立的项目空状态。空状态沿用联系人和消息页面的布局样式，占满右侧剩余区域并水平、垂直居中文案。

## 非目标

- 不改变项目点击后的详情内容。
- 不持久化上一次选择的项目。
- 不改变搜索或筛选行为。
- 不新增前端测试。

## 验证

- 首次进入项目页面时列表无选中项，右侧显示“选择一个项目查看详情”。
- 点击任意项目后，该项目显示选中态，右侧显示对应详情。
- 运行前端类型检查、Lint 和构建。
