# MVP 需求

日期：2026-07-01

## 一句话范围

第一版先做一个私有部署的企业聊天平台：管理员创建用户，用户登录后能在通讯录看到其他用户，聊天列表里有内置 AI 助手，用户可以和助手对话，也可以和其他用户一对一聊天。

## 明确不做

- 不开放用户注册。
- 不做公开 SaaS。
- 不做群聊。
- 不做外部 IM 接入。
- 不做邮件接入。
- 不做任务管理。
- 不做 Agent 平台。
- 不做复杂组织架构。

这些能力都可以作为后续版本。

## 用户与登录

- 系统安装后应有一个管理员账号。
- 管理员可以在管理端创建用户。
- 用户不能自行注册。
- 用户由管理员分配初始密码或登录凭证。
- 用户可以登录客户端使用产品。
- 用户基础信息包括：姓名、账号、头像、状态。

## 通讯录

- 通讯录展示平台内所有用户。
- 用户可以从通讯录选择某个用户发起一对一聊天。
- 第一版通讯录只包含内部用户。
- 后续再扩展 Agent、外部 IM 联系人、组织部门。

## 聊天列表

- 每个用户的聊天列表默认包含一个内置 AI 助手。
- 用户与其他人发生过对话后，对方出现在聊天列表中。
- 聊天列表区分：
  - AI 助手会话。
  - 用户一对一会话。

## 与 AI 助手对话

- 每个用户都有一个内置 AI 助手入口。
- 用户可以向助手发送文本消息。
- 助手可以返回文本回复。
- 第一版助手只做普通问答。
- 暂不要求助手调用工具、发送消息、管理 todo 或控制其他 Agent。

## 用户一对一聊天

- 用户可以向另一个用户发送文本消息。
- 双方可以看到聊天历史。
- 支持基本消息状态即可，例如已发送、发送失败。
- 第一版不要求已读回执、撤回、引用回复、表情、附件、语音或视频。

## 管理端

管理员至少需要能：

- 创建用户。
- 查看用户列表。
- 禁用用户。
- 重置用户密码或登录凭证。

第一版管理端不需要复杂角色权限。只要区分管理员和普通用户。

## 部署

- 独立部署。
- 不依赖 SaaS 账号体系。
- 可以先用 Docker Compose 安装。
- 服务端保存用户、会话和消息。
- 客户端通过服务端登录和收发消息。

## 后续扩展方向

第一版跑通后，再扩展：

- 通讯录加入 Agent。
- 任务 / todo 面板。
- AI 助手代发消息。
- 外部邮件接入。
- 飞书、钉钉、企业微信等外部 IM 接入。
- 群聊和话题。
- Agent 平台。
- 权限、审计和敏感数据策略。
